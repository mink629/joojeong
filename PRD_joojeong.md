# PRD: 주정 (JOOJEONG)

> 술을 이제 막 즐기기 시작한 2030 주린이가, 위스키·와인·전통주를 아우르며 라벨 사진 한 장으로 시음 기록을 남기고 자신의 맛/향 취향을 데이터로 발견해가는 앱

---

## 논의 흐름 요약 (의사결정 로그)

1. **최초 문제 정의**: 애주가가 재구매·재선택 시 겪는 의사결결 비용 문제를 다양한 각도로 브레인스토밍 (기억 휘발성, 재구매 판단, 취향 미인지, 컬렉션 욕구, 추억 손실, 도구 공백, 공유 어려움)
2. **핵심 가치 확정**: "나만의 시음 노트를 축적해 재구매·재선택 의사결정 비용 단축"으로 1차 스코프 결정
3. **입력 방식 논의**: 신규 유저의 입력 장벽을 낮추기 위해 "라벨 사진 촬영/업로드 → AI 인식 → 자동 채움"을 핵심 기능으로 우선 배치
4. **텍스트 검색 실패 리스크 논의**: 외국어 술 이름 오타로 인한 기록 분산 우려 → 자동완성/유사도 매칭 및 술 마스터 DB 분리 아이디어 논의했으나, 가벼운 MVP 취지에 맞지 않아 **Won't로 이동** (사용자 DB 성장 후 고도화 과제로 로드맵에 남김)
5. **Must 축소**: "라벨 인식+기록 생성/조회" 2개로 좁혀 가벼운 MVP로 1차 확정
6. **B2B 확장 검토**:
   - 재고 태깅(소매점 대상) → 도메인 자체가 달라져 무겁다고 판단, 제외
   - 시음노트 → 메뉴 추천 운영 툴(바/와인바 대상) → 기존 기록 구조를 손님 관점으로 재해석하는 자연스러운 확장으로 채택, **별도 Phase 2 로드맵**으로 유지
7. **방향 대전환 — "재구매 판단" → "취향 발견"**:
   - 타겟을 "숙련 애주가"에서 **"주린이(위스키/와인/전통주 입문자)"**로 변경
   - 배경: MZ세대는 대량음주 문화는 줄었지만, 취향을 탐색하며 세련되게 즐기는 문화는 여전히 존재한다는 관점
   - 감각 태그(다중 선택) 도입, 취향 리포트 기능 신설
8. **감각 태그 정교화**: 단순 4개 태그(달콤함/스모키/상큼함/고소함)로는 위스키·와인의 실제 풍미 스펙트럼을 못 담는다고 판단 → 주종별 태그 세트(7~8개씩)로 확장
9. **스코프 확대 결정**: "가벼운 MVP"로는 원하는 완성도가 안 나온다고 판단 → **조금 무거운 MVP로 전환**. 전통주를 초기 스코프에 정식 포함 (크래프트 맥주는 보류)
10. **재구매 의사 복귀**: "취향 좋아요" 같은 애매한 척도 대신, **명확한 별도 항목(재구매함/보통/비추천)**으로 재도입 — 취향 태그(발견용)와 재구매 의사(판단용)는 목적이 다르므로 함께 유지
11. **가격 항목 복귀**: 최초 스코프에 있었던 가격 입력을 최종적으로 재포함

## 1. 한 줄 정의

술을 이제 막 즐기기 시작한 주린이가, 위스키·와인·전통주를 아우르며 라벨 사진 한 장으로 시음 기록을 남기고 자신의 맛/향 취향을 데이터로 발견해가는 앱

## 2. 문제와 타깃

- **문제**: 요즘 20~30대는 대량 음주 문화 대신 취향을 탐색하며 세련되게 술을 즐기고 싶어하지만, "내가 어떤 맛을 좋아하는지"를 알아갈 도구가 없다. 특히 전통주는 관심은 늘고 있는데도 이를 기록할 도구 자체가 거의 없는 공백 지대다.
- **타깃**: 위스키/와인/전통주를 이제 막 접하기 시작한 20대 후반~30대 초반 직장인. "나는 무슨 맛을 좋아하지?"라는 질문에 아직 답하지 못하지만, 술을 유행이 아닌 자기 취향의 영역으로 즐기고 싶어하는 사람.

## 3. 핵심 시나리오

1. **진입**: 새로운 술을 접함 → 앱 실행
2. **입력**: 라벨 사진 촬영/업로드 (인식 실패 시 이름 텍스트 입력) → 주종 확인/선택(위스키/와인/전통주)
3. **AI 처리**: 브랜드/제품명/종류 인식 → 입력폼 자동 채움 → 주종에 맞는 감각 태그 세트 노출
4. **결과**: 별점 + 주종별 감각 태그 다중 선택 + 재구매 의사 + 가격 + 메모 입력 → 저장
5. **다음 행동**: 기록이 쌓이면 "내 취향 리포트"에서 주종별 선호 패턴 확인 → 다음 술 선택 및 재구매 판단 시 참고

## 4. 기능 범위

**Must**

- 라벨 사진 촬영/업로드 → AI로 브랜드/제품명/종류 인식, 실패 시 이름 텍스트 입력
- 주종 선택(위스키/와인/전통주) 및 주종별 감각 태그 세트 제공
- 시음 기록 생성/조회 (별점, 감각 태그 다중 선택, 재구매 의사, 가격, 메모)
- 내 취향 리포트 (주종별 누적 태그 빈도 기반 선호 패턴 요약)

**Won't (이번에 안 하는 것들)**

- 크래프트 맥주, 사케, 브랜디 등 추가 주종 (향후 확장 후보)
- 전문 시음 노트(수치화된 산도/탄닌/도수별 상세 분석 등 완전한 아로마 휠)
- 텍스트 자동완성/유사도 매칭, 술 마스터 DB 분리 (사용자 DB 성장 후 고도화 과제)
- 소셜 공유, 팔로우, 타인 기록 열람
- AI 기반 취향 추천("이런 술도 시도해보세요")
- 위치 기반 매장 추천, 실시간 시세 비교, 다국어, 오프라인 지원, 게이미피케이션
- B2B 확장(바/와인바 손님 취향 관리 — 별도 로드맵으로 유지)

## 5. 화면 구성

- **화면 1 · 홈(내 술장)**: 최근 기록 리스트(카드형, 주종 배지 + 감각 태그 표시), 주종별 필터 탭(전체/위스키/와인/전통주), "+" 버튼
- **화면 2 · 사진 인식 결과**: 촬영 사진 미리보기, AI 인식값(브랜드/이름/종류), 인식된 주종 확인/수정, 인식 실패 시 텍스트 입력창
- **화면 3 · 기록 작성**: 자동 채움된 이름/브랜드/주종(수정 가능) + 별점 + 주종별 감각 태그 칩(다중 선택) + 재구매 의사(3지선다) + 가격 + 메모, 저장 버튼
- **화면 4 · 내 취향 리포트**: 주종별 탭 전환, 각 주종 내 태그 빈도 시각화, 가장 많이 선택된 태그 하이라이트
- **화면 5 · 술 상세**: 기록 상세(재구매 의사/가격 포함), 수정/삭제

## 6. AI 동작 정의

- **입력**: 라벨 사진 1장 (jpg/png, 최대 5MB)
- **출력**: `{ "brand": string, "product_name": string, "type": "위스키" | "와인" | "전통주" | null, "confidence": number }`
- **프롬프트 방향**: 라벨 이미지에서 브랜드/제품명과 주종(위스키/와인/전통주)을 추출. 확신 없으면 confidence 낮게, 추측 대신 빈 값 반환. 임의로 정보를 지어내지 않음. 주종 판별이 애매하면(예: 증류식 소주 vs 위스키) 사용자가 직접 선택하도록 null 반환.
- **실패 처리**: 인식 실패/저신뢰도 시 → 안내 문구 + 수동 입력 폼(이름/브랜드/주종 선택) 노출

## 7. 기술 스택

Next.js(React) + Tailwind, Supabase(Postgres + Storage + Auth), Claude API(이미지 인식), Vercel 배포

## 8. 데이터

## 9. 마일스톤과 완료 기준

- [ ] M1: 더미 데이터로 사진→주종 확인→감각태그 선택(위스키 세트만 우선)→재구매의사/가격 입력→기록 저장 흐름 동작
- [ ] M2: 실제 이미지 인식 API 연동, 와인/전통주 태그 세트 추가, 주종별 동적 태그 로직 완성
- [ ] M3: 내 취향 리포트(주종별 집계) 구현
- [ ] M4: 배포 및 실사용자 테스트

**완료 기준**: 신규 유저가 주종을 넘나들며 5~10개 정도 기록을 쌓았을 때, 취향 리포트로 자신의 선호 패턴을 발견하는 동시에, 특정 술의 재구매 여부와 가격대를 즉시 확인할 수 있다.

## 향후 확장 로드맵 (참고용, 이번 스코프 아님)

- 크래프트 맥주, 사케 등 추가 주종
- 감각 태그 → 전문 시음 노트(아로마 휠 수준)로 세분화
- 텍스트 자동완성/유사도 매칭, 술 마스터 DB 분리 (사용자 DB 규모 확대 시 재검토)
- 취향 리포트 기반 AI 추천
- B2B 확장: 바/와인바 직원이 손님의 취향 리포트를 보고 맞춤 추천 (Phase 2)
  - 손님 프로필 생성/검색, 손님별 시음 기록(AI 인식 엔진 재사용 + 손님 반응/메모), 손님 상세 히스토리 조회
